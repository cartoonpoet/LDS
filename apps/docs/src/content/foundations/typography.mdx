<h2 id="type-scale">타입 스케일</h2>

v3 문서와 컴포넌트는 업무형 제품에 맞춘 실용적인 타입 체계를 전제로 합니다.

- 섹션 진입에는 디스플레이/헤딩 레벨을 사용합니다
- 폼과 컨트롤 라벨은 중간 굵기를 기본으로 사용합니다
- 운영 화면 본문은 밀도는 높되 읽기 가능성을 유지합니다

<h2 id="font-stack">폰트 스택</h2>

기본 폰트 스택은 `Pretendard`를 우선하고, 이후 `Segoe UI`, 마지막으로 일반 sans-serif로 내려갑니다.

<h2 id="practical-rules">운영 규칙</h2>

- 라벨은 짧고 직접적으로 유지합니다
- 데이터 밀도가 높은 화면에서는 장식적인 굵기를 피합니다
- 운영성 상세 설명은 헤딩이 아니라 본문 스타일을 사용합니다
